# 基于 RDK X5 的智能管道巡检机器人与交互式状态反馈系统

> 端-边-云协同的燃气泄漏检测系统 — 自主行走、AI 检测、服务端告警、网页展示，全链路闭环。

---

## 1. 项目简介

### 背景

化工园区管道气体泄漏是一大隐患。传统人工巡检效率低、存在安全风险，现有自动化方案误报率高、缺乏云端实时反馈。

本系统基于地平线 RDK X5 边缘计算平台，结合 Ackermann 机器人小车、热成像相机和自建服务端，设计了一套端-边-云协同的智能巡检方案。

### 系统架构

```
小车(STM32) ──串口──▶ RDK X5 ──HTTP──▶ 服务器 ──▶ Web 仪表盘(:5002)
                  │                      │             │
                  ├ 热红外相机(640×512)   ├ 数据存储     ├ 登录门户(:5001)
                  ├ 采集卡 MS210x        └ 告警分发     └ 实时监控+语音告警
                  └ 三级 AI 检测管道
```

### 核心特性

- **自定义路径巡航** — 结构化指令语言（f/t/s），里程计闭环修正，一行命令定义完整巡检路线
- **三级级联 AI 检测** — CV 四通道融合发现 → MobileNetV2 精确分类 → VLM 语义确认，算力按需调度
- **实时 Web 仪表盘** — 6 场景点状态卡片、MJPEG 实况预览、语音告警播报
- **端-边-云协同** — 板端采集推理 → 服务端存储分发 → 网页实时展示，全链路数据互通

## 3. 使用方法

### 自定义路径巡检

```bash
python3 main.py --route "f:0.1,s f:0.2,t:-15,s f:0.5,s" --loc 1
```

路径命令语言：

| 命令 | 含义 | 示例 |
|------|------|------|
| `f:X` | 前进 X 米 | `f:0.3` |
| `t:X` | 转向 X 度（正左负右） | `t:-90` |
| `s` | 定点检测（停 1s → 录制 → 停 1s） | `s` |
| `s:X` | 定点检测，录制 X 秒 | `s:3` |

机器人行驶至每个检测点位自动停车录制热成像画面，数据送入三级融合算法完成识别，结果实时回传。

### 离线检测

```bash
# npz 录制文件
python3 main_0.py --npz record_xxx.npz --mode bg_subtract --loc 1

# 视频输入
python3 main_0.py --input video.mp4 --mode bg_subtract

# 图片目录
python3 main_0.py --input 图片目录/ --mode bg_subtract
```

### 仪表盘操作

浏览器访问 `localhost:5001` → 登录 → 进入操作控制台。

- **左侧**：MJPEG 实时预览 + 巡检指令输入 + 终端日志面板
- **右侧**：6 个场景点状态卡片 + 检测结果图片
- **告警**：检测到泄漏时自动语音播报（"场景点X，有气体泄漏，请注意！"× 2）

### 实验效果

搭建三类典型化工园区场景，还原真实巡检工况：

| 场景 | 判定 | 系统行为 |
|------|------|---------|
| 静态空景 | normal | 正常显示 |
| 模拟气体泄漏 | danger | 标记告警 + 联动语音播报 |
| 人员实体干扰 | warning | VLM 识别干扰源，抑制误报 |

## 4. 技术原理

### 三级级联检测架构

```
原始热力图 (640×512 灰度)
     │
     ▼
┌─────────────────────────────────────────────────────┐
│ 第一级：CV 信号发现                                    │
│ MOG2背景减除 + 自适应帧差 + LK光流 + Welford方差        │
│     ↓ 四通道加权融合为热力图，互补覆盖、天然抗噪          │
└─────────────────────────────────────────────────────┘
     │ 检出有效 ROI
     ▼
┌─────────────────────────────────────────────────────┐
│ 第二级：DL 精确分类                                    │
│ MobileNetV2 轻量卷积神经网络                            │
│ → 算力按需调度，整体算力消耗降低 60%                      │
│ → 模型体积 9 MB，单帧推理 < 0.1s                        │
│ → 输出：leak / normal 二分类                            │
└─────────────────────────────────────────────────────┘
     │ CV 与 DL 结论冲突
     ▼
┌─────────────────────────────────────────────────────┐
│ 第三级：VLM 语义确认                                    │
│ Qwen-VL-Max 视觉大模型对热成像截图进行语义级分析           │
│ → 消除人员走动、手部遮挡等 CV/DL 无法判定的模糊场景        │
└─────────────────────────────────────────────────────┘
     │
     ▼
  最终分级输出：danger / warning / normal
```

### CV 四通道融合

| 通道 | 方法 | 作用 |
|------|------|------|
| 背景减除 | MOG2 | 检测相对背景的变化区域 |
| 帧差 | 自适应阈值帧差法 | 捕获帧间扩散信号 |
| 光流 | 稀疏 LK 光流追踪 | 跟踪气体运动方向 |
| 时域统计 | Welford 方差分析 | 捕捉气体边缘不稳定性 |

四个通道加权融合为单张热力图，实现互补覆盖与天然抗噪——单通道易受光照、震动等干扰，四通道联合大幅降低误触发。

### 判定逻辑

```
第一级 CV → 无异常 → normal

第一级 CV → 检出 ROI → 第二级 MobileNetV2 分类
    ├─ 高置信 leak  → danger（触发语音告警）
    ├─ 高置信 normal → normal
    └─ 无法判定      → 第三级 VLM 语义确认
                         ├─ 确认为气体 → danger
                         └─ 确认为干扰 → warning
```

### 模型

| 项目 | 详情 |
|------|------|
| 架构 | MobileNetV2 |
| 训练数据 | GOD-Video ~28 万张热红外灰度图 |
| 输入 | 224×224（灰度三通道复制） |
| 输出 | leak / normal 二分类 |
| 模型体积 | 9 MB |
| 单帧推理 | < 0.1s（RDK X5 CPU） |
| 验证准确率 | 98.7% |

### 端云协同

板端（RDK X5）完成采集与推理后，将热成像实时画面、算法判定结果、可视化特征图上传至服务端，同步推送前端网页实时展示。小车 → 板子 → 服务器 → 网页控制台，全链路数据互通，远程即可掌控巡检现场。
